# Student Learning Collections by Competency

Version: 07082026 v1

Use this page when you know the skill you need but not the day number. Each collection points back to the day folders and current student-facing materials.

## Output evidence and exact visible results

Use these when you need to prove what the program actually shows.

- Day 1: [Visible output evidence](week01/day01_visible_output_evidence/) — Use printed output as evidence instead of guessing what code did.
- Day 8: [Debug/test design](week02/day08_debug_test_design/) — Use expected-versus-actual evidence and protective tests to diagnose behavior.
- Day 18: [Synthesis workbook](week04/day18_synthesis/) — Combine planning, tracing, testing, debugging, and explanation routines.

## State, variables, and values

Use these when the question depends on what value is stored now or what changes over time.

- Day 2: [Variables, expressions, and state](week01/day02_variables_expressions_state/) — Track what value is stored now after assignment and expression evaluation.
- Day 11: [For loops, range, and counters](week03/day11_for_loops_range_counters/) — Trace repeated state changes pass by pass.
- Day 12: [While loops, update, and termination](week03/day12_while_loops_update_termination/) — Prove that repeated work eventually stops.
- Day 16: [List mutation and indexed processing bridge](week04/day16_list_mutation_indexed_processing/) — Update list state intentionally and verify the changed list.

## Types, casting, truthiness, and boundaries

Use these when code behavior depends on value kind, Boolean logic, or a cutoff.

- Day 3: [Types, casting, and truthiness](week01/day03_types_casting_truthiness/) — Check value type before trusting calculation, comparison, or condition behavior.
- Day 4: [Comparisons and boundaries](week01/day04_comparisons_boundaries/) — Reason about exact thresholds, boundary cases, and comparison outcomes.
- Day 5: [Boolean combinations + Week 1 checkpoint/closeout practice](week01/day05_boolean_combinations_and_week1_closeout/) — Combine several true/false conditions and use the checkpoint as a low-stakes readiness check.
