## What you are learning today

- Focus: Branch outcomes.
- Main evidence to keep: condition order, selected branch, condition evidence.
- Why it matters: Identify which branch outcome occurs and justify it from condition evidence.

## Before class / during class / after class

- Before class: Preview the ordered conditions before evaluating them.
- During class: Trace from the top and stop at the first branch whose condition is true.
- After class: Write the selected branch and the evidence that selected it.

## If you get stuck

If stuck, ask which condition is checked first and whether it is true.

Try to name the kind of evidence you need before asking for help: output, state, type, branch decision, test, loop pass, list state, or support plan.

## How this connects to ENGR 102

This prepares students to explain program behavior in engineering decisions with multiple branches.
