## What you are learning today

- Focus: Ordered validation.
- Main evidence to keep: validation order, rejected cases, accepted branch.
- Why it matters: Preserve rule order and reject invalid evidence safely.

## Before class / during class / after class

- Before class: List the validation rules in order.
- During class: Check invalid or risky cases before normal cases.
- After class: Name one case that would fail if the order changed.

## If you get stuck

If stuck, trace the same case through the first two rules only.

Try to name the kind of evidence you need before asking for help: output, state, type, branch decision, test, loop pass, list state, or support plan.
