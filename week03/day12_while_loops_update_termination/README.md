## What you are learning today

- Focus: While loops, update, and termination.
- Main evidence to keep: loop condition, update, termination reason.
- Why it matters: Prove that repeated work eventually stops.

## Before class / during class / after class

- Before class: Find the condition and the update before tracing.
- During class: Check whether the update moves toward stopping.
- After class: Write why the loop stops or why it risks not stopping.

## If you get stuck

If stuck, ask what changes before the next condition check.

Try to name the kind of evidence you need before asking for help: output, state, type, route, test, loop pass, list state, or support plan.

## How this connects to ENGR 102

This supports safe loop design and debugging in regular-semester problems.
