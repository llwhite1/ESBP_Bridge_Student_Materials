## What you are learning today

- Focus: Break/continue and loop debugging.
- Main evidence to keep: break point, continue point, skipped action, final state.
- Why it matters: Distinguish skipped work from stopped work.

## Before class / during class / after class

- Before class: Identify whether the code can skip work or stop early.
- During class: Mark where control flow jumps during the loop.
- After class: Write whether work was skipped, stopped, or completed normally.

## If you get stuck

If stuck, trace one pass at a time and label the control-flow action.

Try to name the kind of evidence you need before asking for help: output, state, type, route, test, loop pass, list state, or support plan.

## How this connects to ENGR 102

This helps students debug loops where the final answer depends on control flow, not just arithmetic.
