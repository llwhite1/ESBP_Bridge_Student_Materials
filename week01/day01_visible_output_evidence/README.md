## What you are learning today

- Focus: Visible output evidence.
- Main evidence to keep: final output, visible traces, exact wording.
- Why it matters: Use printed output as evidence instead of guessing what code did.

## Before class / during class / after class

- Before class: Skim the working rules and mark where output evidence appears.
- During class: Write down exact output before explaining it.
- After class: Circle one output line that proves your answer.

## If you get stuck

If stuck, rerun the code mentally line by line and ask: what is visible now?

Try to name the kind of evidence you need before asking for help: output, state, type, route, test, loop pass, list state, or support plan.

## How this connects to ENGR 102

This starts the ENGR 102 habit of making claims from evidence, not confidence.
